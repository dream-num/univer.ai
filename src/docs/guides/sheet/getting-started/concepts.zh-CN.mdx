# 基本概念

<BadgeGroup values={[UniverTypes.SHEET, UniverTypes.DOC, UniverTypes.SLIDE]} value={UniverTypes.SHEET} />

在进一步深入使用 Univer 之前，请阅读本篇文档以了解 Univer 的一些基本概念，这将有助于你更顺利地使用 Univer。

基本概念包括：

* 插件
* 快照
* 命令
* Facade API

## 插件（Plugin）

Univer 的功能都是由插件提供的，插件是 Univer 的基本组成单元，多个插件组合在一起，就构成了一个 Univer 应用。插件的职责包括：

1. 提供电子表格、文档或者幻灯片的某种功能
2. 提供基础能力
3. 

插件使得你能够按照自己的需要组合 Univer 的功能，从而定制出适合自己的 Univer 应用，同时避免加载多余的代码，提升性能。请按照你的实际需要在项目中引入 Univer 插件。

当然你也可以通过引入 UMD 包来引入目前 Univer 提供的所有第一方功能（不过这样就失去了性能方面的收益），或者是通过 Univer CLI 来根据你的需求定制一个 UMD 包。

## 快照（Snapshot）

想要在 Univer 中加载文档，你需要在 Univer 中加载快照。每种类型的文档都提供了自己的快照格式：

1. 电子表格的快照格式是 `SheetSnapshot`
2. 文档的快照格式是 `DocSnapshot`
3. 幻灯片的快照格式是 `SlideSnapshot`。

## 命令（Command）

在 Univer 中，对于值的修改都需要通过命令执行，基于命令的方式可以更好地追踪值的变化，并且能够实现撤销、重做和协同编辑等功能，以及处理功能之间复杂的关联逻辑等等。**请不要通过赋值属性的方式修改 snapshot！**，直接修改值不会触发视图或者 UI 的更新。请通过命令或者 Facade API 来修改值。
