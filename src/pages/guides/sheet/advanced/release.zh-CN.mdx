# 其他发布版本

> 体验最新的功能和特性，同时帮助我们测试和改进 Univer。

除了稳定版本之外，Univer 还提供了 alpha 和 nightly 通道。这些版本可以让你提前体验到尚未正式发布的最新功能。但请记住，更多的功能同时意味着更多的风险。这些版本可能包含错误、不完整的功能或者不稳定的特性。请尽可能避免在生产环境中使用这些版本。

## Alpha Release

当 Univer 开发团队完成了一个新功能或重大更改时，可能会将其预发布到 alpha 通道，你可以通过以下命令安装 alpha 版本的 Univer 包：

<Tabs items={['pnpm', 'npm']}>
  <Tabs.Tab label="pnpm">
    ```shell
    pnpm add @univerjs/<package-name>@alpha
    ```
  </Tabs.Tab>
  <Tabs.Tab label="npm">
    ```shell
    npm install @univerjs/<package-name>@alpha
    ```
  </Tabs.Tab>
</Tabs>

## Nightly Release

夜间版本是 Univer 自动构建的版本，每天都会生成最新的更改、错误修复和实验性功能。这些版本非常不稳定，不适合在生产环境中使用。但它们非常适合测试最新的功能和确保与最新的代码库兼容。

为了降低用户无意中安装夜间版本的风险，我们并未在 npm 上发布该类型的版本。但你可以通过以下步骤在你的项目中尝试它：

首先，你需要在项目的根目录下创建一个名为 `.npmrc` 的文件，然后在该文件中添加以下内容：

```Properties
@univerjs:registry=https://verdaccio.univer.work/
```

这个配置将 `@univerjs` 作用域指向了 Univer 的夜间版本仓库，该仓库托管在 [https://verdaccio.univer.work/](https://verdaccio.univer.work/)。这是我们提供的一个专门用于测试和开发的私有 npm 仓库。

然后，你可以使用以下命令安装 Univer 包的夜间版本：

<Tabs items={['pnpm', 'npm']}>
  <Tabs.Tab label="pnpm">
    ```shell
    pnpm add @univerjs/<package-name>@nightly
    ```
  </Tabs.Tab>
  <Tabs.Tab label="npm">
    ```shell
    npm install @univerjs/<package-name>@nightly
    ```
  </Tabs.Tab>
</Tabs>

特别需要重申的是，夜间版本是非常不稳定的，可能包含致命的错误、不完整的功能或者实验性的特性。请**绝对**不要在生产环境中使用这些版本。

如果你在使用夜间版本时遇到任何问题或者有任何反馈，请通过适当的渠道向 Univer 开发团队报告，比如 Github Issues 或者 Discord 社区。

祝你在抢先体验中玩得愉快！
